Hugo博客搭建

时间： 2022-02-16 | 字数： 2317 字 | 阅读： 5分钟 | 阅读次数：

前面已经很快速的实现了WordPress博客框架的搭建，搭建过程非常简单，甚至没有手动进行过配置，并没有深刻体会到网站开发的过程。

基于热爱挑战热爱学习、无穷无尽折腾的原则，也想换一个WordPress里没有的主题nexT，我决定选择hugo来再一次搭建我的博客。

本文将从以下几个方面讲解hugo网站搭建过程：

  • Hugo简介及预备工作
  • 主题安装并本地测试
  • 修改配置及内容

Hugo简介及预备工作

据了解，目前流行的个人博客框架有很多，各有其特点，比如Hexo、Hugo、WordPress等等，WordPress在前面我们已经了解过，而Hexo与Hugo是较为相似的两个框架，在经过对比之后，发现Hugo的本地环境配置较为简单快速，所以我选择了Hugo作为本网站的框架。

  • 安装hugo：

Hugo，一种用 Go 语言编写的静态网站生成器。简单、易用、高效、易扩展、快速部署。 它号称是世界上最快的框架。官网为： https://gohugo.io。 在中文官网中我们可以方便地查阅帮助文档 ，即：https://www.gohugo.org

我的操作系统是Windows，所以安装时前往 https://github.com/gohugoio/hugo/releases 页面，下载 hugo_X.XX_Windows-64bit.zip，解压即可得到对应的Hugo二进制文件Hugo.exe

  • 安装Git：

Git 是一个开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。Git 不仅仅是个版本控制系统，它也是个内容管理系统(CMS)，工作管理系统等。

Windows安装：进入网站https://git-scm.com/下载安装

  • 安装Go1.4+：

即Go语言的安装，安装地址：The Go Programming Language，并设置好GOPATH环境变量

主题安装并本地测试

用Hugo创建第一个新网站

首先准备新建一个文件夹用于存放网站文件，例如：E:\Myblog\test

在cmd中切换至本路径，然后执行下方代码

hugo new site quickstart

上述操作会在E:\Myblog\test文件夹中创建一个名为quickstart的文件夹，整个网站的文件都在这个文件夹中，如theme里面存主题文件，content里面存markdown文件

接下来在cmd中进入该路径：E:\Myblog\test\quickstart 并输入：

git init
git clone --recurse-submodules https://github.com/elkan1788/hugo-theme-next.git

第一行命令是初始化站点的所有文件

第二行命令是克隆了GitHub上的nexT主题

下载好之后将hugo-theme-next目录下的config和content拷贝到站点根目录quickstart下

此时该站点已经配置完成，接下来在cmd中执行下方代码生成站点服务：

hugo server

该命令就是启动了本地服务器，使用浏览器打开网址http://localhost:1313/或127.0.0.1就可以预览，如下图所示：

至此就已经完成了nexT主题的安装，并在本地成功打开了正常的网页，接下来我们要根据自己的需求将网页修改为自己的个人博客。

修改配置和内容

毫无疑问，这一部分是最耗费时间的一部分，前面的安装和测试按照教程步骤来，就算遇到小问题也可以及时解决。

但是想要修改配置信息，需要穿梭于每个文件夹里的每个配置文件，搞清楚每一个配置文件有什么作用，对应的是哪一块的内容，举个例子：在config文件夹中的config.toml负责一些基本配置，比如主题的选择、语言的选择、timeout的设定，emoji或math是否启用等等；menu.toml负责菜单栏的设定，可以按个人意愿增删改，并修改样式；params.toml负责界面中的很多元素设定，比如社交网站（如Github）入口，页脚Footer的设定，博客作者的头像昵称和介绍的修改等等。在hugo-theme-next文件夹中的配置文件更多且更为复杂，此处暂不做解释。

所以在修改配置方面所需工作量较大，也难以解释，在此处就不再赘述，以下主要讲解新建一篇文章的注意事项：

新建markdown文章

新建文章均在content文件夹下的post文件夹里，可以直接把markdown文件拷贝过去，也可以用命令行新建一个文件：

hugo new post/test.md

自动生成的md文件中包含最基本的配置内容，如下所示：

+++
date = "2015-10-25T08:36:54-07:00"
draft = true
title = "test"

+++

正文内容

其中两个+++之间是文章的发布时间、标题、是否为草稿等设定，这些内容是toml格式的，也可以换成YAML格式（用两个---来标记），还有更多的设定如下所示：

+++
author:"作者昵称"
title: "文章标题"
description: "文章的描述信息"
tags: [ "标签1", "标签2", "标签3" ]
categories: [ "分类1", "分类2" ]
keywords: [ "Hugo", "keyword" ]
date: 2012-04-06

# 排序你的文章
weight: 40

#打开数学公式渲染
math = true
+++

这句话将展示在主页上

正文内容

正文内容可用其他软件撰写，如Wolai、语雀等等笔记软件，导出为markdown文件后，设定好配置配置内容，拷贝到post文件夹中，就可以发布在网页。
